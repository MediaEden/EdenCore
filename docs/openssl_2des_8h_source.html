109/*
110 * This is the DES encryption function that gets called by just about every
111 * other DES routine in the library. You should not use this function except
112 * to implement 'modes' of DES. I say this because the functions that call
113 * this routine do the conversion from 'char *' to long, and this needs to be
114 * done to make sure 'non-aligned' memory access do not occur. The
115 * characters are loaded 'little endian'. Data is a pointer to 2 unsigned
116 * long's and ks is the DES_key_schedule to use. enc, is non zero specifies
117 * encryption, zero if decryption.
118 */
